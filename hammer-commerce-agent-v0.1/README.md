# Hammer Commerce Agent V1.0

**Autonomous Commerce Agent（自主商业 Agent）**。

它不再定位为选品软件。产品目标是让用户尽量只说一句目标，由 Agent 连续完成商业任务，并且只在登录、发布、下单、支付等必要的安全与合规节点请求确认。

## 最高开发原则

> 每新增一个功能，都必须减少用户至少一次操作。

每个 Sprint 必须同时提交：

1. 改造前人工操作数
2. 改造后人工操作数
3. 实际减少的操作数

不能减少人工操作的页面、图表、评分或装饰性功能暂不开发。必要的安全确认不视为失败，但必须做到一次确认、自动恢复，不让用户重复输入。

## V1.0 首项减操作改造

任务链缺少候选商品时，原流程需要跳转页面、分别填写商品名称/成本/售价/运费/平台费、提交分析、返回任务链并手动恢复。

现在只需要：

```text
粘贴一句商品资料
  → Agent 自动提取字段
  → 自动分析利润与风险
  → 自动保存商品
  → 自动恢复原任务链
```

示例：

```text
桌面风扇，成本15，售价39.9，运费5，备注夏季小件
```

验收结果：从至少 7 项表单/导航操作减少到“粘贴一句 + 确认一次”共 2 项，至少减少 5 次人工操作，没有增加新页面。

## 自主任务链

```text
用户目标
  → 寻找候选商品
  → 检查利润是否可卖
      └─ 不合格：自动换下一个候选
  → 生成商品资料
  → 准备图片任务
  → 准备发布
      └─ 等待主人确认 / 后续 Browser Agent
  → 等待成交结果
  → 记录真实利润
  → 生成今日汇报
```

状态：

```text
WAITING → RUNNING → BLOCKED → RUNNING → SUCCESS
                         └────────────→ FAILED
```

`BLOCKED` 表示 Agent 已完成当前能自动完成的工作，正在等待必要条件，不代表失败。

## 核心模块

```text
src/core/chain-planner.js              连续任务规划
src/core/chain-executor.js             自动推进、重试与断点恢复
src/core/chain-store.js                任务链持久化
src/core/sales-store.js                成交与利润记录
src/tools/chain-tools.js               任务链工具集合
src/tools/product-quick-capture-tool.js 一句话商品资料解析工具
```

能力统一通过 Tool Registry 注册，为 Browser、OCR、图片识别、价格识别、物流、通知、文件、Excel 和长期记忆工具保留扩展位置。

## 当前真实边界

Browser Agent 尚未接入，因此暂时不能自动登录、搜索或发布到真实电商平台。需要外部平台操作时任务链会暂停并等待确认，不会虚构搜索、发布或成交结果。

下一阶段只开发能减少操作的 Browser Agent：自动打开公开页面、搜索、翻页、记录价格、截图和汇总报告。涉及账号登录与真实发布时遵守平台规则并保留授权确认。
