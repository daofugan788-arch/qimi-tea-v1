# Hammer AI V0.8

面向普通微信卖货商家的移动端网页版内容助手。

首页价值说明：告诉 Agent 你卖什么，1分钟生成朋友圈、小红书、短视频卖货内容。

核心流程：

```text
打开网页 → 告诉 Agent 商品 → Agent 主动追问 → 生成/连续修改 → 一键复制
```

## V0.8 手机卖货 Agent

- 首页直接进入对话式 Agent，不需要先理解表单
- 主动询问商品名称、价格、卖点、客户和发布平台
- 支持“更口语、短一点、强调价格、换平台”等连续修改
- 同一篇内容可以持续修改，不重复计算生成次数
- 在当前浏览器记住商品、常用平台和最近任务
- 再次打开时可直接继续上次商品
- 保留原有表单入口和 V0.7 付费验证闭环
- 默认本地模式支持限定修改；接入正式模型后支持自由语言改写

## V0.7 付费验证闭环

- 第一次内容生成免费，不阻挡新商家体验
- 第二次生成前展示商家内测方案
- 默认内测价为 19.9 元/月，可在 `config.js` 修改
- 完整流程：方案页 → 收款页 → 订单编号 → 已付款 → 等待人工确认
- 支持复制订单编号和发送订单给管理员
- 本地记录生成、复制、方案查看、收款页和付款申请次数
- `paymentQrUrl` 留空时自动进入安全测试模式，不会发生真实扣款
- 配置正式收款码后使用收款码 + 人工确认，暂不接账号、数据库和复杂支付网关

## 配置收款码

在 `config.js` 的 `PAYMENT_CONFIG.paymentQrUrl` 填入公开可访问的收款码图片地址。未填写时，按钮明确显示“模拟已付款（测试）”，适合先验收完整购买流程。

## 验收目标

- 陌生商家 5 秒内知道产品用途
- 不需要注册或学习提示词
- 1 分钟内完成第一次内容生成
- 结果可以一键复制发布
- 手机浏览器直接打开使用

## 支持内容

- 微信朋友圈成交文案
- 小红书标题与体验分享
- 抖音 30 秒口播脚本
- 淘宝详情卖点结构

## 本地运行

ES Modules 需要通过 HTTP 打开：

```bash
python3 -m http.server 8787
```

然后访问：

```text
http://127.0.0.1:8787
```

## Vercel 部署

将本目录设为 Vercel 的 Root Directory，Framework Preset 选择 Other，保持默认构建设置即可上线。

## AI 配置

默认使用本地模板，打开即可体验，商品资料不会上传。

`config.js` 保留 OpenAI 兼容接口配置。正式上线时必须通过服务端代理保存 API Key，不要把真实密钥写入静态网页或公开仓库。

## 冻结边界

本版本不包含自动发布微信、抖音账号管理、视频剪辑、AI 管家、商城系统和用户社区。
